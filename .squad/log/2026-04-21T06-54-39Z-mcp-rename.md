## Problem

The directory `agents/mcp/` had the same name as the installed `mcp` PyPI package (v1.27.0). When Python was run from inside `agents/`, the current working directory on `sys.path` caused `import mcp` to resolve locally instead of to the installed package, breaking `import mcp.server.fastmcp`.

## Solution

Renamed `agents/mcp/` → `agents/mcp_servers/` via `git mv` to preserve history. Updated all references in `.vscode/mcp.json` and `agents/README.md`.

## Outcome

✅ **Fixed.** Module import now works from any directory. Verified on Python 3.10+.
